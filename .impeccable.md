## Design Context

### Users

TerraTech mod users are the primary audience. The product should bias toward first-time users who need help getting mods working fast and reliably, while still staying efficient for experienced users who already understand collections, validation, and launch flow.

The primary job to be done is managing mods in one place with minimal friction: configure the game and mod manager once, resolve problems quickly when they occur, and then get out of the way so the user can launch TerraTech without extra ceremony.

### Brand Personality

The interface should feel utilitarian, dense, and lightweight.

It should project reliability and directness more than personality or spectacle. Users should feel that the app is a practical tool they can trust, not a destination they linger in.

### Aesthetic Direction

This should look like a focused desktop utility for managing a game setup, not a social media product and not a content-discovery surface. The app should not mimic a Workshop browser, feed, storefront, or promotional landing page.

Visual emphasis should go to status clarity, configuration confidence, and fast action routing. Dense information is acceptable as long as hierarchy remains obvious and the app stays easy to scan.

The target surface is a desktop Electron app for a PC game. Mobile phone layouts are not a product goal; responsive work should focus on supported desktop window sizes, the Electron minimum window, keyboard/mouse workflows, and readable dense tables.

### Design Principles

1. Prioritize fast, reliable setup and recovery over visual novelty.
2. Make primary actions obvious for first-time users without slowing expert repetition.
3. Keep the interface dense but disciplined: compact, scannable, and low-noise.
4. Use feedback to build trust, especially around validation, save, dependency, and launch flows.
5. Design for desktop mod-management workflows, not mobile consumption.
6. Remove anything that feels like browsing, promotion, or discovery rather than management.
